🎮 Simulation & What-If Service

Công cụ mô phỏng kịch bản - Thử nghiệm các quyết định hạ tầng trong môi trường ảo của AegisFlow AI

Mục Đích​

Simulation Service (Dịch vụ Mô phỏng) cho phép các nhà quy hoạch và quản lý đô thị thử nghiệm hàng trăm kịch bản khác nhau trước khi triển khai thực tế trên nền tảng AegisFlow AI:

  1. Mô phỏng Kịch bản (What-If)

    • Xây dựng tuyến đường mới hoặc cầu vượt
    • Thay đổi các nút giao thông và tín hiệu đèn
    • Mô phỏng tác động của các công trình công cộng mới (bệnh viện, trường học)
    • Thay đổi chính sách phân luồng giao thông

  2. Dự báo Tác động Đa chiều

    • 📊 Kinh tế: Dự báo thay đổi GDP vùng, việc làm và khả năng tiếp cận thương mại
    • 🌍 Môi trường: Tác động đến CO₂, chất lượng không khí và rủi ro ngập úng mới
    • 👥 Xã hội: Chỉ số bình đẳng, khả năng tiếp cận y tế và giáo dục
    • 🚗 Giao thông: Giảm thiểu ùn tắc, thời gian di chuyển và an toàn lộ trình

  3. So sánh Kịch bản (A/B Testing)

    • So sánh trực quan giữa các phương án quy hoạch
    • Đánh giá Impact Score (Điểm tác động) tương đối
    • Hỗ trợ lựa chọn giải pháp tối ưu dựa trên dữ liệu bằng chứng

Quy Trình Mô Phỏng​

1. Khởi tạo Kịch bản (Scenario)
   └─ Kéo-thả hạ tầng trên bản đồ AegisFlow
   └─ Thiết lập tham số (ngân sách, khung thời gian, mục tiêu)

2. Nhân bản Trạng thái GIS & Map
   └─ Lấy snapshot trạng thái thực tế của đô thị từ Digital Twin

3. Áp dụng Thay đổi (Apply Changes)
   └─ Cập nhật đồ thị mạng mạng lưới (Network Topology)
   └─ Thêm/sửa các thuộc tính hạ tầng

4. Chạy Mô phỏng Hành vi (Agent-Based Model)
   └─ Mô phỏng di chuyển của hàng triệu agent (người dân)
   └─ Đánh giá tương tác với hạ tầng mới

5. Dự báo Tác động với AI
   └─ Phân tích ngữ cảnh với Amazon Nova
   └─ Trích xuất chỉ số tác động kinh tế - xã hội - môi trường

6. Tổng hợp Báo cáo & Impact Score
   └─ Tính toán điểm tổng hợp (0–100)
   └─ Vẽ biểu đồ Radar và biểu đồ xu hướng theo thời gian
   └─ Giải thích kết quả bằng ngôn ngữ tự nhiên (AI Generated)

7. Lưu trữ & Chia sẻ
   └─ Lưu vào cơ sở dữ liệu để so sánh
   └─ Xuất báo cáo PDF cho các bên liên quan

API Endpoints​

Quản lý Kịch bản​

# Tạo kịch bản mới
POST /api/scenarios
{
  "name": "Cải tạo nút giao thông A",
  "changes": [
    { "type": "new_road", "geometry": "..." }
  ]
}

# Lấy kết quả mô phỏng
GET /api/scenarios/{id}/results

Chạy Mô phỏng​

# Bắt đầu quá trình mô phỏng
POST /api/scenarios/{id}/simulate

Response:
{
  "status": "processing",
  "eta_seconds": 180,
  "simulation_id": "sim_987"
}

📊 Phân Tích & Dashboard Backend​

Ngoài khả năng mô phỏng kịch bản tương lai, dịch vụ này còn cung cấp các API phân tích sâu cho Dashboard của AegisFlow AI.

🔍 Tổng hợp Dữ liệu Đa chiều​

  • Phân tích Chuỗi thời gian: Tổng hợp dữ liệu ngập lụt và giao thông theo giờ/ngày/tháng.
  • Phát hiện Bất thường: Tự động nhận diện các mẫu dữ liệu lạ từ cảm biến IoT.
  • Phân tích Nhóm: So sánh hiệu suất xử lý sự cố giữa các quận/huyện.

📈 Loại báo cáo hỗ trợ​

  • Báo cáo Vận hành: Tổng quan sức khỏe đô thị thời gian thực.
  • Báo cáo Hiệu suất: Đánh giá tốc độ phản ứng khẩn cấp (SLA).
  • Bản đồ Nhiệt (Heatmap): Tích hợp dữ liệu GIS để hiển thị mật độ sự cố và rủi ro.

🔗 Tích hợp Hệ thống​

Dịch vụ Simulation & Analytics kết nối chặt chẽ với các thành phần khác:

  • GIS & Map Service: Lấy dữ liệu nền không gian (Baseline State).
  • AI Prediction Service: Sử dụng các mô hình dự báo rủi ro ngập để đưa vào kịch bản.
  • IoT Service: Nhận dữ liệu thực tế qua Kafka để hiệu chỉnh mô hình mô phỏng.
  • Notification Service: Gửi thông báo khi quá trình mô phỏng hoàn tất hoặc phát hiện báo cáo bất thường.

Deployment​

Dịch vụ được đóng gói dưới dạng Docker container và có khả năng scale ngang bằng Ray Cluster để xử lý các mô phỏng kịch bản phức tạp.

docker-compose up -d simulation-service

📊 Cấu trúc dữ liệu ClickHouse​

Bảng Events​

CREATE TABLE events (
    event_id UUID,
    event_type String,
    timestamp DateTime,
    user_id UInt64,
    entity_id UInt64,
    entity_type String,
    metadata String,
    created_at DateTime DEFAULT now()
) ENGINE = MergeTree()
ORDER BY (timestamp, event_type);

Tổng hợp báo cáo​

CREATE MATERIALIZED VIEW reports_daily AS
SELECT
    toDate(created_at) as date,
    category,
    status,
    count() as total_reports,
    avg(resolution_time) as avg_resolution_time
FROM reports
GROUP BY date, category, status;
